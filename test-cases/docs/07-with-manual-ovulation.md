# 수동 배란일 입력 테스트 케이스

## 📌 공통 입력 조건

모든 테스트 케이스는 동일한 입력 조건을 사용합니다:

- **생리 기록**: 
  - 2025-01-01 ~ 2025-01-05
  - 2025-01-29 ~ 2025-02-02
  - 2025-02-26 ~ 2025-03-02
- **배란 테스트 기록**: 없음
- **배란일 직접 입력**: 
  - 2025-01-15 (사용자가 직접 입력한 배란일)
  - 2025-02-12 (사용자가 직접 입력한 배란일)
  - 2025-03-12 (사용자가 직접 입력한 배란일)
- **피임약 패키지 시작일**: 없음
- **오늘 날짜**: 2025-03-15
- **생리 주기 설정**: 
  - 수동 평균 주기: 28일
  - 수동 평균 기간: 5일
  - 자동 계산 사용: false
- **피임약 설정**:
  - 피임약으로 계산: false

**⚠️ 주요 차이점**: 사용자가 배란일을 직접 입력한 경우
- 배란기: 사용자 입력 날짜 (특정 날짜)
- 가임기: 배란일 -2일 ~ +1일 (사용자 입력 기준)
- 입력이 없는 주기는 기본 공식 사용

---

## TC-07-01: 1일 조회
### 📌 테스트 목적
특정 날짜 1일만 조회할 때 수동 배란일 입력이 반영되는지 검증

### 📝 입력 조건
- **조회 기간**: 
  - 시작: 2025-04-15
  - 종료: 2025-04-15

### ✅ 예상 결과
- **생리 예정일들**: 없음 (조회 기간 내 생리 없음)
- **생리 지연일**: 없음
- **배란기들**: 없음 (조회 기간 내 배란기 없음)
- **가임기들**: 없음 (조회 기간 내 가임기 없음)

### 🔍 검증 포인트
- [ ] 1일 조회 시 해당 날짜만 반환되는가
- [ ] 수동 배란일 입력이 계산에 반영되는가

- **배란기들**:

- **가임기들**: 
  - 2025-04-08 ~ 2025-04-11 (조회 범위 04-08 ~ 04-14와 부분 겹침, 배란일 -2 ~ +1)

## TC-07-03: 1개월 조회 (생리 사이)
### 📌 테스트 목적
실제 기록된 두 생리 기간 사이의 구간을 조회할 때 수동 배란일 입력이 반영되는지 검증하고, 다음 생리 정보도 함께 제공되는지 확인

### 📝 입력 조건
- **조회 기간**: 
  - 시작: 2025-02-03
  - 종료: 2025-02-25

### ✅ 예상 결과

**주기 개수: 2개**

**주기 1 (pk=2):**
- **실제 생리 기록**: 2025-01-29 ~ 2025-02-02 (두 번째 생리)
- **생리 예정일들**: 없음 (Period 3는 실제 기록, 예정일이 아님)
- **생리 지연일**: 없음
- **배란기들**: 
  - 2025-02-12 ~ 2025-02-12 (사용자 직접 입력 날짜)
- **가임기들**: 
  - 2025-02-10 ~ 2025-02-13 (배란일 -2 ~ +1)
- **주기**: 28일

**주기 2 (pk=3):**
- **실제 생리 기록**: 2025-02-26 ~ 2025-03-02 (세 번째 생리, 다음 주기 정보 제공)
- **생리 예정일들**: 없음 (조회 범위 밖)
- **생리 지연일**: 없음
- **배란기들**: 없음 (조회 범위 밖)
- **가임기들**: 없음 (조회 범위 밖)
- **주기**: 28일

### 🔍 검증 포인트
- [ ] 사용자 직접 입력 배란일이 우선 적용되는가 (pk=2)
- [ ] 가임기가 사용자 입력 기준으로 조정되는가 (pk=2)
- [ ] 기본 공식 대신 사용자 입력을 사용하는가 (pk=2)
- [ ] 다음 생리 정보(pk=3)가 함께 반환되어 주기의 끝을 알 수 있는가
- [ ] pk=3의 배란기/가임기는 조회 범위 밖이므로 비어있는가

## TC-07-04: 3개월 조회
### 📌 테스트 목적
장기간 조회 시 수동 입력이 있는 주기와 없는 주기가 구분되는지 검증

### 📝 입력 조건
- **조회 기간**: 
  - 시작: 2025-03-01
  - 종료: 2025-05-31

### ✅ 예상 결과
- **실제 생리 기록**: 2025-02-26 ~ 2025-03-02 (pk=3, 마지막 생리)
- **생리 예정일들**: 
  - 2025-03-26 ~ 2025-03-30
  - 2025-04-23 ~ 2025-04-27
  - 2025-05-21 ~ 2025-05-25
- **배란기들**: 
  - 2025-03-12 ~ 2025-03-12 (사용자 직접 입력)
  - 2025-04-10 ~ 2025-04-10 (사용자 입력 패턴 기반 예측)
  - 2025-05-08 ~ 2025-05-08 (사용자 입력 패턴 기반 예측)
- **가임기들**: 
  - 2025-03-10 ~ 2025-03-13 (배란일 -2 ~ +1, 사용자 입력 기반)
  - 2025-04-08 ~ 2025-04-11 (사용자 입력 기반)
  - 2025-05-06 ~ 2025-05-09 (사용자 입력 기반)
- **주기**: 28일

### 🔍 검증 포인트
- [ ] 사용자 입력이 있는 주기는 입력값 우선인가
- [ ] 다음 주기 예측이 사용자 입력 패턴을 고려하는가
- [ ] 가임기가 배란일 -2 ~ +1일로 일관되게 계산되는가
